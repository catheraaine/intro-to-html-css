Anatomy of a website

Concrete example

  • A paragraph is your content.

  • Putting your content into an HTML tag to make it look like a paragraph is structure.

  • 
                <p>A paragraph is your content.</p>
  • Making the font of your paragraph blue and 18pt is presentation:

    A paragraph is your content.

HTML Tags

  • Our HTML lives are made up of tags.
  • All HTML tags have < and >
  • There are a variety of tags: p, strong, div, em, span, section, etc.
    "etc." is not one...
  • A paragraph in HTML is simply: 
    
<p>There's a tag on the left and a tag on the right.</p>

Anatomy of an HTML Element

  • Element
    • An individual component of HTML
    • Paragraph, Table, List
    • <strong>This is an HTML element, right here</strong>

Anatomy of an HTML Element, cont.

  • Tag
    • Markers that signal the beginning and end of an element
    • Opening tag and Closing Tag 
      
<tagname>Stuff in the middle</tagname>

    • 
 <p>This is a sample paragraph.</p>

Anatomy of an HTML Element, cont.

  • Container Element
    • An element that can contain other elements or content
    • A paragraph (<p>) contains text
  • Stand Alone Element
    • An element that cannot contain anything else
    •  <br>
 <img> 
 <hr>
    • Just drop it in your code.

Anatomy of an HTML Element, cont.

  • Attribute
    • Each element can have a variety of attributes
    • Language, style, identity, source, link
    • They look like:
      name="some value here"
  • Drop them 'inside' your opening HTML tag
    • Value is the value assigned to a given attribute.
    • 
 <a href="http://catherinerust.com">Catherine's Website</a>
 <img src="cats.jpg"/>

Doctype

The Ô¨Årst thing on an HTML page is the "doctype", which tells the browser which version of the markup language the page is using.

Here's the doctype we use to tell the browser "Hey, this is HTML5, buddy."


          <!DOCTYPE html>

HTML Tag

After <doctype>, the page content must be contained between <html> tags.


            <!DOCTYPE html>
            
            <html>

              All of our content is going to live between 
              these two tags.

            </html>

Head and Body Tags

The <head> contains meta (internal) information about the page.

The <body> contains the actual content of the page.

Head and Body Tags

This is the basis of a complete HTML web page, called a skeleton.


            <!DOCTYPE html>
            <html>

               <head>
                <title>Title of the page</title>
               </head>

               <body>
                The page content here.
               </body>

            </html>

Inline vs Block

So far, we have mostly seen block elements

They begin on the next line, like paragraphs.



There are also inline elements

They appear on the same line that they were written on.

Formatted text


<p>
  Here is a paragraph with <em>Emphasized</em> text and <strong>Important</strong> text.
</p>

Here is a paragraph with Emphasized text and Important text.



Notice: <em> and <strong> are meant to indicate meaning through style. If you want to have italicized or bold text for appearance - and not to communicate meaning - you should use CSS.

Images

Images also have three components

  • Tag: <img/>
  • src attribute: location of your image
  • alt attribute: description for screen readers
  • Let's all try it! Copy this code into your HTML.

<img src="http://www.fillmurray.com/350/200" alt="Bill Murray"/>

Images, Cont.

  • On a "real" website, we would be using local images - typically stored in an "images" subfolder in the same place as your .html file.

  • This would look more like:

<img src="/images/billmurray.png" alt="Bill Murray"/>
